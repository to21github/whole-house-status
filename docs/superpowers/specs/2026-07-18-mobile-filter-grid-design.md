# 移动端房间筛选网格设计

## 目标

让移动端的房间筛选区域使用与统计卡片相同的内容宽度，消除右侧无意义的深色留白，并将圆形显示图标放在最后一个房间按钮的右侧，而不是独占下一行。

## 布局

- 仅在 `max-width: 920px` 的移动端断点生效；桌面端筛选栏保持现有横向弹性布局。
- `.filter-bar` 在移动端使用三列等宽网格，宽度由 `.page` 的左右内边距约束，因此右边界与统计卡片右边界一致。
- 房间导航不再建立独立网格容器，而是将其按钮作为筛选栏网格项参与排版。
- 房间按钮填满各自网格列，保留现有文字大小、最低高度、圆角和选中样式。
- `.display-menu` 是房间按钮之后的下一个网格项：当“未分组”位于最后一行第一列时，显示图标位于第二列；第三列可以为空，但不会产生筛选区域右侧的额外留白。
- 显示菜单继续以图标自身为定位锚点，展开后的“显示已忽略的实体”选项仍限制在移动端视口内。

## 实现边界

- 不改动房间解析、排序、选中状态或显示已忽略实体的业务逻辑。
- 不移动 HTML 结构或修改 JavaScript 渲染逻辑；移动端通过 CSS `display: contents` 让现有房间按钮和显示菜单成为同一个网格的项。
- 不调整统计区、实体卡片、按钮文字大小或桌面端布局。

## 测试策略

- 更新现有移动端房间控制测试，验证筛选栏为三列网格并占满可用内容宽度。
- 验证每个房间按钮的实际列宽相同，首列跨行对齐。
- 验证显示图标与“未分组”处于同一行且位于其右侧，并保留现有移动端菜单视口边界测试。
- 运行完整 `npm run verify`，确认单元测试与全部 Playwright 前端测试通过。

## 非目标

- 不改变房间按钮在桌面端的固定宽度或显示图标的圆形尺寸。
- 不将显示图标改为文字按钮。
- 不变更 Add-on 配置 schema 或 Home Assistant 实体数据。
