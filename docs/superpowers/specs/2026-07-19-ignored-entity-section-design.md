# 已忽略实体独立卡片区设计

## 目标

勾选“显示已忽略的实体”后，将当前房间内的已忽略实体卡片独立显示在未忽略异常卡片之前，避免已忽略的超时、高功率、离线或故障实体与需要处理的异常混在一起。

## 渲染顺序

- 筛选栏之后首先显示已忽略实体卡片区；该区不添加标题。
- 仅当“显示已忽略的实体”已勾选，且当前房间存在已忽略实体时，显示该卡片区和其后的分隔线。
- 已忽略卡片区包含所有有效的已忽略实体，包括原本位于 `alerts` 中的异常实体与原本位于 `devices` 中的普通实体。
- 已忽略卡片继续沿用现有“已忽略”提示、降低透明度和“不再忽略”操作。
- 下方异常卡片区只显示未忽略的超时、高功率、离线或故障实体。
- 普通设备卡片区只显示未忽略的普通实体。

## 房间和状态处理

- “全部”与具体房间均复用当前房间过滤规则；切换房间后，三类卡片区同时更新。
- 尚未勾选显示已忽略实体时，已忽略卡片在所有分区中隐藏。
- 用户在面板中忽略或恢复实体时，使用现有的 `effectiveIgnored` 状态；勾选显示时卡片会立即在三个分区间移动，无需等待 Home Assistant 响应。
- 统计数字、Home Assistant 实体可见性和后端 ViewModel 数据结构保持不变。

## 实现边界

- 在页面中增加隐藏状态的 `#ignored` 卡片容器和紧随其后的分隔线。
- 在前端渲染中基于 `effectiveIgnored` 把已通过房间过滤的 `alerts` 与 `devices` 划分为已忽略和未忽略列表。
- 继续复用 `renderCards` 与现有卡片样式，不增加标题、额外配置项或后端接口。

## 测试策略

- 扩展 Playwright 测试，使用同一房间中的已忽略异常、未忽略异常、已忽略普通设备和未忽略普通设备。
- 验证未勾选时已忽略卡片区和卡片均隐藏。
- 验证勾选后已忽略异常与普通实体均只出现在 `#ignored`，未忽略异常只在 `#alerts`，未忽略普通设备只在 `#devices`。
- 切换房间后验证三个区均按当前房间过滤。
- 运行完整 `npm run verify`。

## 非目标

- 不添加“已忽略的实体”标题。
- 不改变异常判断、统计口径、房间导航或实体忽略的存储方式。
- 不变更 Home Assistant Add-on 配置 schema。
