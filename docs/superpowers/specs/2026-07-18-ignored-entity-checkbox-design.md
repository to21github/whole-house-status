# 显示已忽略实体复选框设计

## 目标

将“显示已忽略的实体”选项左侧的浏览器原生复选框改为与参考图一致的紧凑方框，同时保留现有显示/隐藏逻辑、键盘操作和本地持久化行为。

## 交互与视觉

- 复选框视觉尺寸固定为 `18px × 18px`。
- 未选中状态：透明背景、浅灰色 `1px` 描边、轻微圆角。
- 选中状态：浅灰色填充、深色对勾。
- 键盘聚焦时显示可见焦点外框，不依赖鼠标悬停。
- 点击方框或“显示已忽略的实体”文字都继续切换原生 checkbox 的 `checked` 状态。
- 现有 `aria-label`、本地存储键和显示逻辑保持不变。

## 实现边界

- 保留原生 `<input type="checkbox">` 作为真实交互控件，确保键盘、表单语义和屏幕阅读器行为不变。
- 在 input 后增加一个仅用于视觉呈现的方框元素；通过相邻选择器根据 input 的 `:checked` 和 `:focus-visible` 状态更新样式。
- input 采用可访问的视觉隐藏方式，不使用 `display: none` 或移除焦点能力。
- 仅修改显示选项的 HTML/CSS 和对应前端回归测试，不改变实体过滤、忽略操作或移动端菜单布局。

## 测试策略

- 在现有 Playwright 前端测试中增加未选中状态断言：视觉方框为 `18px × 18px`，透明底色和浅灰描边。
- 增加选中状态断言：方框具有浅灰填充，并存在对勾伪元素。
- 保留现有“勾选后显示、取消后隐藏”的行为测试，确保结构调整没有改变功能。
- 运行完整 `npm run verify`，同时检查工作区不把 `.playwright-cli/` 生成目录纳入提交。

## 非目标

- 不调整“显示”圆形按钮尺寸或房间导航。
- 不修改 Home Assistant 实体隐藏状态、统计数字或实体卡片样式。
- 不增加新的设置项或配置 schema。
